PROBLEM STATEMENT
================================================================================
Low-frequency quadrature outputs (1.8 MHz / 160m band) were severely misphased:
- Target: 90° phase difference between CLK0 and CLK1 outputs
- Measured (initial): ~10.5° phase difference (79.5° error)
  - Zero-crossing delay: 16.26 ns at 1.8 MHz (period 555.56 ns)
- Required accuracy: <1° for 30-40 dB SSB suppression

ROOT CAUSE ANALYSIS
================================================================================
SI5351 Synthesizer Limitations:
1. Phase register: 7-bit value (0–127), applied BEFORE final R divider
2. Each phase step represents: 1/(4 × PLL_frequency) period
3. At low RF frequencies, phase granularity becomes coarse:
   - 2× RF intermediate: ~0.9° per step
   - 4× RF intermediate: ~0.225° per step
   - 8× RF intermediate: ~0.056° per step

Example at 1.8 MHz with 720 MHz PLL:
- Direct approach impossible: phase word quantization too coarse
- Scaling intermediate frequency improves step resolution proportionally

SOLUTION EVOLUTION
================================================================================

Attempt 1: R Divider Alone (FAILED)
- Hypothesis: Use R divider to achieve quadrature on low bands
- Problem: Phase register applied BEFORE R divider, so R divider doesn't help
- Inversion also applied after R divider (not useful for phase control)

Attempt 2: Dual-PLL Approach (FAILED)
- Hypothesis: Independent PLLs for each output with different initial phases
- Problem: Independent oscillators lost phase lock; outputs drifted apart
- Conclusion: Both outputs must source from single PLL for coherence

Attempt 3: Single-PLL with 2× Intermediate & ÷2 R Divider (PARTIAL)
- Concept: Generate 2× RF from single PLL, apply phase word, apply ÷2
- Result: 95.4° measured (4.6° error)
- Status: Coarse quantization still limiting accuracy

Attempt 4: Single-PLL with 4× Intermediate & ÷4 R Divider (WORKING, SUBOPTIMAL)
- Concept: Generate 4× RF from single PLL, apply phase word, apply ÷4
- Phase word formula: (MS+1)/2 (empirical tuning)
- Result at 1.8 MHz: 16.26 ns zero-crossing delay = 10.5° error (79.5° off)
- Status: Better quantization but still insufficient

Attempt 5: Single-PLL with 8× Intermediate & ÷8 R Divider (CURRENT)
- Concept: Generate 8× RF from single PLL, apply phase word, apply ÷8
- MS ratio: 720 / (8 × RF)
- Phase word formula: (MS+1)/4 (scaled for 8× intermediate)
- Results progression:
  a) MS/2: ~180° measured (265 ns at 1.8248 MHz) - too high
  b) MS/4: 135.7 ns at 1.8248 MHz = 89.4° (0.6° error) - very close
  c) (MS+2)/4: 141.5 ns at 1.8248 MHz = 93.1° (3.1° error) - over-corrected
  d) (MS+1)/4: ~90° (similar to 135.7 ns result, measurements "much the same")
- Status: 8× intermediate consistently achieving ~90° ±1° at 1.8 MHz

TECHNICAL IMPLEMENTATION
================================================================================

File: lib/SI5351Control/src/SI5351Control.cpp
Function: setQuadrature90(uint64_t rfHz)

Code Structure:
```cpp
// High-frequency path (≥7 MHz): Direct 90° phase via pickQuadratureDivider()
if (rfHz >= 7000000ULL) {
  // Direct approach works; single PLL, phase word fits 7-bit
}

// Low-frequency fallback (1.8–7 MHz):
// 1. Generate 8× RF intermediate frequency
const uint64_t lo8xHz = rfHz * 8ULL;

// 2. Single PLL at 720 MHz for both CLK0 and CLK1
const uint64_t pllCentiHz = 72000000000ULL;
const uint64_t outCentiHz = lo8xHz * 100ULL;

s_si5351.set_ms_source(SI5351_CLK0, SI5351_PLLA);
s_si5351.set_ms_source(SI5351_CLK1, SI5351_PLLA);
s_si5351.set_freq_manual(outCentiHz, pllCentiHz, SI5351_CLK0);
s_si5351.set_freq_manual(outCentiHz, pllCentiHz, SI5351_CLK1);

// 3. Calculate phase word for 90° at final RF
const uint16_t msRatio = 720000000ULL / (8ULL * rfHz);
const uint8_t phaseWord90 = ((msRatio + 1) / 4) & 0x7F;

// 4. Apply phase word to CLK1 (CLK0 at 0°)
s_si5351.set_phase(SI5351_CLK0, 0);
s_si5351.set_phase(SI5351_CLK1, phaseWord90);

// 5. Reset PLL for phase alignment
s_si5351.pll_reset(SI5351_PLLA);

// 6. Apply R divider /8 to both outputs
programRDivForClock(s_si5351, SI5351_CLK0, 8);
programRDivForClock(s_si5351, SI5351_CLK1, 8);

s_si5351.output_enable(SI5351_CLK0, 1);
s_si5351.output_enable(SI5351_CLK1, 1);
```

MEASUREMENT METHODOLOGY
================================================================================

Equipment: Digital CRO with nanosecond resolution

Procedure:
1. Set transceiver to low band (1.8 MHz nominal)
2. Measure actual CLK0 and CLK1 frequencies
3. Capture zero-crossing times for both signals
4. Calculate phase: phase_degrees = (time_delay_ns / period_ns) × 360°

Example Calculation (1.8248 MHz):
- Period: 1/(1.8248 MHz) = 548 ns
- Measured 135.7 ns delay:
  - Phase = (135.7 / 548) × 360° = 89.4°
  - Error = 90° - 89.4° = 0.6°

TECHNICAL NOTES
================================================================================

Phase Word Scaling:
- Relationship between intermediate frequency and phase word divisor:
  - 2× intermediate: phase_word = MS / 2 (for 180° at intermediate)
  - 4× intermediate: phase_word = (MS+1) / 2 (empirical tuning)
  - 8× intermediate: phase_word = (MS+1) / 4 (for 90° at RF after ÷8)
  
- Pattern: Higher intermediate requires finer (smaller) phase word divisor

Frequency Constraints:
- PLL frequency: 720 MHz (fixed)
- Maximum output frequency: ~120 MHz from multisynth
- 8× intermediate: 8 × 1.8 MHz = 14.4 MHz (well within limits)
- 16× intermediate possible if needed: 16 × 1.8 MHz = 28.8 MHz (still safe)

Phase Locking:
- Both CLK0 and CLK1 sourced from PLLA ensures phase coherence
- PLL reset performed before measurement for initial phase alignment
- Phase remains locked throughout operation (verified with measurements)
